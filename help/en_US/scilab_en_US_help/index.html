SciMP

  • SciMP
    • critical — To enable the critical region.
    • destroy_lock — Uninitializes a lock.
    • get_active_level — Returns the number of active nested parallel regions that enclose the task containing the call.
    • get_dynamic — Returns a value that indicates if the number of threads available in upcoming parallel regions can be adjusted by the run time.
    • get_level — Returns the number of nested parallel regions that enclose the task containing the call.
    • get_max_active_levels — This function obtains the maximum allowed number of nested, active parallel regions
    • get_max_threads — Return the maximum number of threads used for the current parallel region that does not use the clause num_threads.
    • get_nested — This function returns a value that indicates if nested parallelism is enabled.
    • get_num_procs — Get the number of processors that are available when the function is called.
    • get_num_teams — Returns the number of teams in the current team region.
    • get_team_num — Returns the team number of the calling thread.
    • get_thread_limit — Get the maximum number of OpenMP threads available to participate in the current contention group.
    • get_thread_num — Get the thread execution sequence.
    • get_total_thread — Get the total number of threads running in the parallel region.
    • get_wtime — Get the elapsed wall clock time.
    • in_parallel — Returns non-zero if this function is called within a parallel region.
    • init_lock — Initialize a simple lock.
    • master — To enable the master region.
    • mat_mul — Returns the product of 2 matrices.
    • omp — To enable the parallel region.
    • set_dynamic — Enable or disable the dynamic adjustment of the number of threads within a process. The function takes the language-specific equivalent of true and false, where true enables dynamic adjustment of team sizes and false disables it.
    • set_lock — Set a simple lock.
    • set_max_active_levels — Limits the maximum allowed number of nested, active parallel regions
    • set_nested — Enables nested parallelism.
    • set_total_thread — Set the total number of threads to be used in the parallel region.
    • single — To enable the single region.
    • team — To enable the teams region.
    • unset_lock — Releases a lock.
